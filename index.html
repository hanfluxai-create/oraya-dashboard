🐛 Active Bug · Inventory Query Token Blow-Up (Apr 18)

Symptom: Inventory query pulls for specific vehicles consume ~8K tokens per call vs 3K baseline. Price field does not load back into the voice agent response despite data being present in the cached inventory JSON.

Impact: Demo risk — if Mazzio / Greg / Teresa asks "what's the price on that GLS?" mid-call, agent stalls or returns price-absent response. Cost risk — sustained 8K-token calls triple Retell spend.

Root-cause hypothesis: (a) inventory JSON bloat being passed in full context window rather than pre-filtered via n8n lookup, (b) price field path mismatch between crawler output and agent prompt template, (c) retrieval node returning raw array instead of single-record object.

Rebuild plan: 1) Add n8n HTTP lookup node — take VIN/model from agent tool call, return single-record object. 2) Trim inventory context to ≤5 vehicles per turn. 3) Explicitly map price, MSRP, out_the_door fields. 4) Add token-count guard in post-call log. 5) Retest against 20-vehicle sample before demo.

Owner: CTO-Claude (rebuild + validation) · ETA: before next Mazzio demo.

Phase 4 · Regulatory Deep Dive · 7 Statutes

1 · TCPA / FCC-24-17 · AI Voice Classification

Core rule: FCC Order 24-17 (Feb 8 2024) classified AI-generated voices as "artificial or prerecorded" under TCPA. ALL outbound AI voice calls require prior express written consent (PEWC).

Vacatur: FCC's "one-to-one consent" rule vacated by 11th Circuit Jan 24, 2025 (day before effective).

Oraya posture: Enforces most conservative standard regardless. Caller-specific PEWC via Supabase consent_log before Retell dials.

Exposure: $500/violation (negligent), $1,500/violation (willful). No cap. Class action exposure.

2 · CIPA §§ 632 / 632.7 · California All-Party Consent

Core: Cal. Penal Code § 632 prohibits recording without all-party consent. § 632.7 covers cellular/cordless. Smith v. LoanMe (2021) confirmed automated calls covered.

Oraya: Hard-coded recording disclosure at call start. Non-consent triggers basic_attributes_only purge via webhook.

Exposure: $5K/violation (§ 637.2). Criminal liability possible (§ 632).

3 · CPRA / CCPA · Data Minimization & Service Provider

Framework: CPRA effective Jan 1 2023. Regs finalized Sep 22 2025, effective Jan 1 2026. Oraya processes data ONLY as "service provider" under § 1798.140(ag).

Oraya: DPA drafted per § 1798.100(d). RLS on consent_log. 30-day auto-purge for non-consented data.

Gap: DPAs must be countersigned with ALL 6 vendors. Drafted, not executed. Strategy Law LLP intake covers this.

4 · CPPA ADMT · Automated Decision-Making Technology
